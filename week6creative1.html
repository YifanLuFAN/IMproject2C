User input, I think this is the place that best reflects the interactive design of a website. We learned about mouse input and sound input this week.
Regarding mouse input, inspired by the website I saw on openprocessing, I also decided to make a website with different feedback for users clicking and moving the mouse.
The website I made will initially have squares that move randomly from the center to the surroundings. When the user clicks the mouse, all the squares will converge on the straight line formed by the center of the screen and the mouse click point. This process requires the user's mouse to stay for a while. If the user starts to move the mouse, the squares will diverge from the click point again, making a "meteor hammer" style pattern.
(Idea got from openprocessing)
